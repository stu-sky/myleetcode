# 238. 除自身以外数组的乘积

**代码**：[codes/0238-product-of-array-except-self.go](../codes/0238-product-of-array-except-self.go)

## 题目

给你一个整数数组 `nums`，返回数组 `answer`，其中 `answer[i]` 等于 `nums` 中**除 `nums[i]` 之外**其余各元素的乘积。

**说明**：题目数据保证 `nums` 中任意元素的全部前缀、后缀（甚至整个数组）的乘积都在 32 位有符号整数范围内。

**进阶**：在 **O(n)** 时间内、**O(1)** 额外空间（**输出数组不算**额外空间）完成。

**示例**：

- 输入：`nums = [1,2,3,4]` → 输出：`[24,12,8,6]`
- 输入：`nums = [-1,1,0,-3,3]` → 输出：`[0,0,9,0,0]`

## 思路

### 知识点：前缀积（和「前缀和」同一类姿势）

**前缀积**就是把数组从左到右（或从右到左）一路乘过去，得到「从头到当前位置」的累积乘积。本题要的是「去掉 `nums[i]` 后剩下所有的乘积」，自然拆成 **`i` 左边乘在一起** 和 **`i` 右边乘在一起** 两块；这两块分别就是**左前缀积在 `i` 之前停一步**、**右后缀积从 `i` 之后开始**，和用前缀和表示「区间和」是同一套分解思想。

### 怎么想到

- **题目在问什么**：每个位置要「全体乘积去掉自己」，若先算总乘积再除以 `nums[i]`，一方面题目常要求**不用除法**，另一方面数组里若有 **0**，除法会失效或需要大量特判。
- **朴素做法卡在哪**：若对每个 `i` 都扫一遍其余元素相乘，是 **O(n²)**。
- **换什么技巧**：把「除自己以外的乘积」写成 **左段乘积 × 右段乘积**，两边各 **O(n)** 预处理或递推即可，整体 **O(n)**；第二遍用**一个变量**滚右段乘积，就能把辅助数组压掉，满足 **O(1) 额外空间**。

### 核心步骤

1. **从左到右**：`answer[i]` 先存 `nums[0] * … * nums[i-1]`（`i` 左侧的乘积），用一个变量 `left` 边扫边乘。
2. **从右到左**：用变量 `right` 表示 `nums[i+1] * … * nums[n-1]`，每步 `answer[i] *= right`，再 `right *= nums[i]`。

两遍结束后，`answer[i]` = 左 × 右，即为所求。

### 复杂度

- **时间**：O(n)，每个元素常数次操作。
- **空间**：O(1) 额外（仅若干变量；长度为 `n` 的 `answer` 为题目要求的输出）。

## 易错点

1. **不要用「总乘积 ÷ nums[i]」** 作为主要解法：违反「不用除法」的常见约束，且 **0** 会导致除零或繁琐分支。
2. **第二趟从右向左时**，应先 `answer[i] *= right`，再更新 `right`，顺序反了会用错「右边还没乘进去的」那一段。
3. **整数溢出**：Go 中用 `int` 提交时注意平台；题目若强调 32 位，需留意语言与评测环境的类型宽度（力扣通常与题面保证一致）。
