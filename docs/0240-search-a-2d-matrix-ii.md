# 240. 搜索二维矩阵 II

**代码**：[codes/0240-search-a-2d-matrix-ii.go](../codes/0240-search-a-2d-matrix-ii.go)

题库入口：[240. 搜索二维矩阵 II](https://leetcode.cn/problems/search-a-2d-matrix-ii/?envType=study-plan-v2&envId=top-100-liked)

## 题目

写一个高效的算法来判断 **`m × n`** 矩阵中是否存在给定整数 `target`。该矩阵具有以下性质：

- 每一行的元素从左到右**升序**。  
- 每一列的元素从上到下**升序**。

**示例**：

- `matrix = [[1,4,7,11],[2,5,8,12],[3,6,9,16],[10,13,14,17],[18,21,23,26]]`，`target = 5` → `true`。  
- 同上，`target = 20` → `false`。

函数签名：`bool` / Go 中为 `func searchMatrix(matrix [][]int, target int) bool`。

## 思路

### 知识点：从拐角出发的「一维二分」阶梯消去

在行、列各自有序的前提下，**左下角（或右上角）那一个数**有一个特点：往**上**走数字变小，往**右**走数字变大（若选右上角则相反：左小下大）。因此可以把当前格当成「比较根」：比 `target` 大就丢掉**整行更下的部分**（上移），比 `target` 小就丢掉**整列更左的部分**（右移），相等则命中。这不是传统意义上的树，但**决策像在一棵隐式 BST 里走一条从根到叶的路径**——每步只走一个分支，且每步**安全地排除一整行或一整列**，不会漏解。

### 怎么想到

- **题目在问什么**：在整体有序结构里判存，希望比「逐行二分」更省。  
- **朴素卡在哪**：暴力 O(mn)；对每行二分是 O(m log n)，没有用上**列向有序**带来的纵轴信息。  
- **换什么技巧**：利用**双方向单调性**，找一个同时「比左/上侧大、比右/下侧小」的角点作起点，把二维比较变成**每步一维消去**。

### 核心步骤（左下角版）

1. 令 `(i, j)` 从 **`(m-1, 0)`** 开始。  
2. 若 `matrix[i][j] == target`，返回 `true`。  
3. 若 `matrix[i][j] > target`，当前列往下只会更大，故 **`i--`**。  
4. 若 `matrix[i][j] < target`，当前行往左只会更小，故 **`j++`**。  
5. 越界仍未命中则 `false`。

（从 **`(0, n-1)`** 出发对称：大则 **`j--`**，小则 **`i++`**。）

### 复杂度

- **时间**：每步 `i` 或 `j` 其一严格推进，最多 **`m + n - 1`** 步 → **O(m + n)**。  
- **空间**：仅若干下标 → **O(1)**。

### 易错点

1. **不要从左上或右下开始**：那两角同时是行、列极值，无法做到「一大一小两个方向」的安全剪枝。  
2. **空矩阵**：`m == 0` 或 `n == 0` 直接 `false`。  
3. 与 [74. 搜索二维矩阵](https://leetcode.cn/problems/search-a-2d-matrix/) 不同：74 是「拉直成一条全局升序」的特例，常对**行首**做二分 + 行内二分；**240 不能假设整表一维有序**，本技巧才成立。
