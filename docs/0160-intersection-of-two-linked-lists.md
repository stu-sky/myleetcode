# 160. 相交链表

**代码**：[codes/0160-intersection-of-two-linked-lists.go](../codes/0160-intersection-of-two-linked-lists.go)

题库入口：[160. 相交链表](https://leetcode.cn/problems/intersection-of-two-linked-lists/?envType=study-plan-v2&envId=top-100-liked)

## 题目

给定两个单链表的头节点 `headA` 与 `headB`，若两链表相交，找出并返回相交的起始节点；若不相交返回 `nullptr` / Go 中为 `nil`。

注意：

- **相交**指两个指针指向链表中同一个节点（内存地址同一），仅用 `next` 走能到达；
- **`Val` 可以重复**，判断是否相交不能以值相等代替节点相同；
- **整个链表不带环**；
- **保证**链表结构在函数返回后不被改写。

函数签名：`getIntersectionNode` / Go 中为 `func getIntersectionNode(headA, headB *ListNode) *ListNode`。

**示例**（结构略）：两链在某一公共节点汇合后共尾；若分叉后不复交 → 返回 `nil`。

## 思路

### 知识点：双指针 +「虚拟拼接」对齐长度差

**双指针**：两个引用各沿一条链表每次前进一步，用来在 \(O(m+n)\) 内对齐信息而不必显式算出长度。**虚拟拼接**：让指针 `a` 先走完 `headA` 再接到 `headB` 的开头继续走，`b` 对称先 `headB` 再 `headA`。**当有交点时**，第一次去交点前两人各走的总步数可写成同一式子 **\(a+b+c\)**（段长含义与推导见下节），故第一次在**同一结点**上会合即交点；**独资段**长短差由「换接对方链头」那段路补齐。**若无交**，两人最终在 **`nil`** 上同时相遇。实现上至多沿两链表各扫一遍量级 → 时间 \(O(m+n)\)。

### 怎么想到

- **题目在问什么**：两段链表分叉后是否在某一节点汇合，本质是找**后缀公共起点**。  
- **朴素卡在哪**：若先数长度再对齐前缀，要写两遍扫描与额外变量；哈希存 A 整条再扫 B **空间 O(m)**。  
- **换什么技巧**：把「长度差」藏进路程——**各走两遍头（自己的 + 对方的）**，让两指针在去交点前走的步数对齐，等价于对齐环外长度差但不显式减法。

### 核心步骤

1. `a`、`b` 分别从 `headA`、`headB` 出发。  
2. **`a != b` 时**：`a` 若未到尾则前进，否则改从 `headB`；`b` 对称前进或改从 `headA`。  
3. 循环结束时 `a` 与 **`b`** 同时为交点或为 `nil`。  
4. 返回 **`a`**（或 `b`，二者相等）。

### 为何在交点汇合：\(a+b+c\) 的推断

（**记号**：以下 \(a,b,c\) 表示**三段链上的结点数**，与上面代码里的指针变量 `a`、`b` 不是同一回事。）

设两链在某一结点相交，且从 **`headA` 到交点**的结点数为 \(a\)，从 **`headB` 到交点**为 \(b\)，从 **交点到表尾（两链共用）**为 \(c\)。则 **`headA` 整条**长 \(a+c\)，**`headB` 整条**长 \(b+c\)。

算法规定：各自走完当前链后接**对方头**再走。

- **走 `headA` 的那根指针**：先在 A 上走 **\(a+c\)**（过交点、一直到 A 的 `nil`），再换到 **`headB`**，走 **\(b\)** 到达交点。总步数 **\((a+c)+b = a+b+c\)**。  
- **走 `headB` 的那根指针**：先在 B 上走 **\(b+c\)**，再换到 **`headA`**，走 **\(a\)** 到达交点。总步数 **\((b+c)+a = a+b+c\)**。

因此两人**第一次同时踩到「交点」这个结点**时，各自都恰好走了 **\(a+b+c\)** 步——不是碰巧「凑满 \(m+n\)」这个式子本身，而是 **\((a+c)+b=(b+c)+a\)** 把两条链的**长短差**消掉了。若本就不相交，可理解为最后在 **`nil`** 上「对称汇合」，同样满足循环结束条件。

（对比：\(m+n=(a+c)+(b+c)=a+b+2c\)，与每人实际走的 **\(a+b+c\)** 不同；口述时记 **「先整链再补对方分叉」** 即得 **\(a+b+c\)**。）

### 复杂度

- **时间**：至多各走两遍 → **\(O(m+n)\)**。  
- **空间**：常量指针 → **\(O(1)\)**。

### 易错点

1. **交点是节点相等**（指针同），不是 **`Val`** 相等。  
2. **无交**时两指针最终会同时落在 **`nil`**，不要写成只看一方为 `nil`。  
3. **有环不适用**本题保证；若有环要先判环或换模型。文末其它主流解法：可先求两链长度、`long` 先走 \(|len_A-len_B|\)，再同步走找相同节点——同阶但更啰嗦。
