# 283. 移动零

**代码**：[codes/0283-move-zeroes.go](../codes/0283-move-zeroes.go)

## 题目

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。必须原地操作，不能复制数组。

**示例**：
- 输入：`nums = [0,1,0,3,12]`
- 输出：`[1,3,12,0,0]`

## 思路

### 问题本质

把所有 0 移到末尾，非零元素相对顺序不变，且**原地**操作。  
不能开新数组，需要一边遍历一边把非零「挤」到前面。

### 解法：双指针

- **left**：下一个非零元素应该放的位置（即「已整理好的非零区」的右边界 +1）
- **right**：遍历指针

当 `nums[right] != 0` 时，说明发现非零，把它和 `nums[left]` 交换，`left++`。这样非零会依次填满 `[0, left)`，剩下的自然都是 0。

### 示例

```
nums = [0, 1, 0, 3, 12]

right=0: nums[0]=0 → 不交换，left 不动
right=1: nums[1]=1 → 与 left=0 交换 → [1,0,0,3,12]，left=1
right=2: nums[2]=0 → 不交换
right=3: nums[3]=3 → 与 left=1 交换 → [1,3,0,0,12]，left=2
right=4: nums[4]=12 → 与 left=2 交换 → [1,3,12,0,0]，left=3

结果：[1, 3, 12, 0, 0]
```

### 复杂度

- 时间 O(n)，空间 O(1)

## 易错点

1. **检测 right 而非 left**：必须判断 `nums[right] != 0`。若判断 `nums[left] == 0`，left 可能一直停在 0 上，无法正确推进。
2. **交换而非覆盖**：用交换保证不丢数；若直接 `nums[left]=nums[right]` 再 `nums[right]=0`，逻辑等价但写法略繁琐。

## 变种思路

| 变种 | 思路 |
|------|------|
| **移动零到前面** | 从右往左扫，right 从末尾开始，left 从末尾开始，非零与 left 交换 |
| **移动指定值到末尾** | 把 `!= 0` 改成 `!= target` 即可 |
| **保持两类元素相对顺序** | 若要把 0 和 1 分开且各自保序，可用两次遍历：先写非零，再填 0 |
