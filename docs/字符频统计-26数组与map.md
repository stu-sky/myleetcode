# 字符频统计：`[26]int` 与 `map[byte]int` 怎么选

本文总结滑动窗口、异位词、子串覆盖等题里常见的**频次数组**写法：何时用长度 26 的数组，何时用 `map[byte]int`（或定长 ASCII 数组）。

---

## 如何判断（看题干对字符的约束）

1. **先读题干**：`s` / `t` 里允许出现哪些字符？是否明确写「仅小写字母」？
2. **仅 `'a'`～`'z'`（26 种）** → 优先 **`[26]int`**，下标 `c - 'a'`。实现简单、无哈希、常数因子小。
3. **含大小写（常见 52 种）**、**含数字或 ASCII 标点**、或题干写「英文字母」但未限定小写 → **`[26]` 不够**。可任选：
   - **`[128]int` 或 `[256]uint8`**：用 `s[i]` 做下标（`byte` 即 `uint8`），覆盖 ASCII；思路仍是「定长频次数组」。
   - **`map[byte]int`**：字符种类少、稀疏时也好写；不手写魔数 128。
4. **Unicode / 任意字符** → 通常用 **`map[rune]int`**（或逐 rune），不要假设 26。

一句话：**字母表大小固定且就是 26 个小写 → `[26]int`；否则用更大定长数组或 `map`。**

---

## 两类写法的共性（题目特征）

这类题往往在问：

- 窗口内某段字符**是否**与另一段**多重集合一致**（异位词、变位词）；
- 或窗口内对某些字符的**出现次数是否 ≥ 目标**（最小覆盖子串）；
- 或「恰好满足某计数模式」的可行子串数量。

共性是：**把「字符 → 次数」当作状态**，用定长数组或哈希表维护，配合双指针 / 滑动窗口，做到线性扫描。

## 和二进制 / 位掩码的关系（扩展）

若题目保证 **26 个小写且每种字符至多出现一次**（如「子串是否为某排列」的频数只有 0/1），有时可用 **26 位位运算** 表示「是否出现过」，但**一旦出现同一字母要计多次**（如 76、一般异位词计数），就要用数组或 map 存**次数**，不能只靠位掩码。
