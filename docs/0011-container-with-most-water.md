# 11. 盛最多水的容器

**代码**：[codes/0011-container-with-most-water.go](../codes/0011-container-with-most-water.go)

## 题目

给定一个长度为 `n` 的整数数组 `height`，每个元素代表一条垂直线的高度。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。返回最大水量。

**面积公式**：`min(height[left], height[right]) × (right - left)`

**示例**：
- 输入：`height = [1,8,6,2,5,4,8,3,7]`
- 输出：`49`（选 height[1]=8 和 height[8]=7，面积 = 7 × 7 = 49）

## 思路

### 问题本质

暴力枚举所有 (left, right) 组合是 O(n²)。需要找到一种方式跳过不可能更优的组合。

### 解法：对撞双指针

从两端开始，每次**移动较矮的一端**。

**为什么移动矮的？**

面积 = 短板 × 宽度。当我们移动一个指针时，宽度一定减 1。那哪个值得移？

- **移动长板**：短板不变（仍是瓶颈），宽度减小 → 面积**一定减小**，毫无意义。
- **移动短板**：可能遇到更高的线，短板可能变大 → 面积**有可能增大**。

所以：**只有移动短板，才有机会找到更大的面积**。这就排除了大量不必要的组合。

### 示例

```
height = [1, 8, 6, 2, 5, 4, 8, 3, 7]
         L                          R

L=0(1) < R=8(7) → 面积 = 1×8 = 8，移动 L
L=1(8) > R=8(7) → 面积 = 7×7 = 49，移动 R
L=1(8) > R=7(3) → 面积 = 3×6 = 18，移动 R
L=1(8) > R=6(8) → 面积 = 8×5 = 40，移动 R
L=1(8) > R=5(4) → 面积 = 4×4 = 16，移动 R
L=1(8) > R=4(5) → 面积 = 5×3 = 15，移动 R
L=1(8) > R=3(2) → 面积 = 2×2 = 4，移动 R
L=1(8) > R=2(6) → 面积 = 6×1 = 6，移动 R
L=1 >= R=1 → 结束

最大面积：49
```

### 复杂度

- 时间 O(n)：左右各走一遍
- 空间 O(1)

## 易错点

1. **移动长板没用**：这是贪心的核心，移动长板面积只可能更小，必须移动短板。
2. **两端相等时**：`height[left] == height[right]` 时移动哪个都行，不影响正确性（不会跳过最优解）。
3. **面积计算**：是 `min(两端高度) × 宽度`，不是 `(左+右) × 宽度`。
