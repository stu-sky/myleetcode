# 239. 滑动窗口最大值

**代码**：[codes/0239-sliding-window-maximum.go](../codes/0239-sliding-window-maximum.go)

## 题目

给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组的最左侧移动到最右侧。你只可以看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。返回每个滑动窗口中的最大值。

**示例**：
- 输入：`nums = [1,3,-1,-3,5,3,6,7], k = 3`
- 输出：`[3,3,5,5,6,7]`

## 思路

### 本题里会用到的两个概念（各说一句）

- **滑动窗口**：长度固定为 `k` 的区间，每次只把右端向右挪一格，左端跟着挪，相当于在数组上「框」着走。本题要对**每一个框住的区间**求最大值。
- **双端队列**：普通队列只能一头进一头出；双端队列**两头都能弹出**。本题需要在**队头**丢掉过期下标、在**队尾**丢掉「不可能再当最大值」的下标，所以用「双端」才方便。（Go 里用切片模拟即可。）

**单调队列**：在双端队列里人为维护一种顺序——本题维护**下标对应的 `nums` 值从队首到队尾单调递减**。这样**队首永远是当前窗口里最大值的候选**，取答案 O(1)。

---

### 怎么想到

1. **题目在问什么**：每个窗口各问一次「这 `k` 个数里谁最大」，共约 `n` 个窗口。
2. **朴素做法**：每滑一次，在窗口里扫一遍找最大 → 约 O(n·k)，数据大时会超时。
3. **卡在哪**：需要**随着窗口右移，尽快丢掉不可能成为答案的数**，而不是每次都重新扫满 `k` 个。
4. **为什么不是前缀和**：区间**和**可以用「前缀和相减」；区间**最大值**没有类似的「两个前缀一减就出来」的公式，思路要换一条。
5. **突破口**：新来的数若比窗口里某些旧数还大，那些旧数在**当前数还在窗口内时**永远当不了最大，可以直接从候选里剔除；同时窗口左边界右移时，下标过期的也要剔除。用**双端队列 + 单调性**同时完成这两类剔除，每个下标进出队列次数有限 → 整体 O(n)。

---

### 怎么做（算法步骤）

队列里只存**下标**（方便判断有没有滑出窗口），且这些下标对应的 `nums` 值**从队首到队尾递减**。

每扫到一个新下标 `i`：

1. **队头过期**：若 `dq[0] <= i - k`，已不在窗口内，从队头弹出。
2. **队尾淘汰**：只要 `nums[dq[队尾]] <= nums[i]`，队尾不可能是当前或之后窗口的最大值，从队尾反复弹出。
3. **入队**：`i` 入队尾。
4. **记答案**：若 `i >= k - 1`，当前窗口最大值为 `nums[dq[0]]`。

---

### 复杂度

- 时间 O(n)：每个下标最多入队、出队常数次
- 空间 O(k)：队列中最多保留与窗口相关的下标

## 易错点

1. **队列存下标、比较用 `nums`**：滑窗靠下标判断，`dq[0] <= i-k` 才成立。
2. **队尾用 `<=` 弹出**：与当前值相等的旧下标也可弹出，保留更靠右的同值即可。
3. **与前缀和区分**：求 max 不用前缀和套路。

## 变种思路

| 变种 | 思路 |
|------|------|
| **滑动窗口最小值** | 改为单调递增（队尾弹出 ≥ 当前值的） |
| **优先队列** | 大根堆存 `(值, 下标)`，懒删过期，O(n log k) |
| **任意区间最值查询** | ST 表、线段树等 |
