# 73. 矩阵置零

**代码**：[codes/0073-set-matrix-zeroes.go](../codes/0073-set-matrix-zeroes.go)

## 题目

给定一个 `m x n` 的矩阵，若某一元素为 `0`，则将其**所在行与整列**的元素都标记为 `0`。请使用**原地**算法（空间复杂度尽量为 O(1)）。

**示例**：

- 输入：`[[1,1,1],[1,0,1],[1,1,1]]`  
  输出：`[[1,0,1],[0,0,0],[1,0,1]]`
- 输入：`[[0,1,2,0],[3,4,5,2],[1,3,1,5]]`  
  输出：`[[0,0,0,0],[0,4,5,0],[0,3,1,0]]`

## 思路

### 知识点：用「首行 / 首列」当标记数组（原地布尔信息）

要在 **O(1) 额外空间** 里记住「哪些行、哪些列该清零」，可以把**第 0 行**当成「列是否该清」的标记条、**第 0 列**当成「行是否该清」的标记条，就像用两块一维数组替代 `m+n` 的额外集合。麻烦在于：**第 0 行、第 0 列自己也可能因原矩阵含 0 而要整行/整列清掉**，所以先用**两个布尔量**单独记下「**原来的**首行 / 首列是否本来就有 0」，避免后面用标记区覆写时把这条信息弄丢。

### 怎么想到

- **题目在问什么**：凡遇 0，抹掉整行整列。
- **朴素卡在哪**：另开 `m×n` 拷贝或 `m+n` 集合最直观，但题目要压空间。
- **换什么技巧**：只有「行 / 列」两种维度，**用第一行记列、第一列记行**即可；首行首列的「自身是否该清」用 **2 个 flag** 单独保存，再按「先内层打标 → 再按标消去 → 最后处理首行首列」口述，面试最好背。

### 核心步骤

1. **读初值**：扫首行得 `firstRow`，扫首列得 `firstCol`（与后面标记无关，专管「最后两行/列要不要全 0」）。
2. **内层打标**：对 `i≥1, j≥1`，若 `matrix[i][j]==0`，则 `matrix[i][0]=0`、`matrix[0][j]=0`（声明第 `i` 行、第 `j` 列待清零）。
3. **根据首列标记清行**：`i=1..m-1`，若 `matrix[i][0]==0`，将第 `i` 行整行置 0。
4. **根据首行标记清列**：`j=1..n-1`，若 `matrix[0][j]==0`，将第 `j` 列整列置 0。
5. **收尾**：若 `firstRow`，第 0 行全 0；若 `firstCol`，第 0 列全 0。

### 复杂度

- **时间**：O(mn)（几遍线性扫描）。
- **空间**：O(1) 额外。

## 易错点

1. **必须先记下 `firstRow` / `firstCol`**，再拿第 0 行、第 0 列去打标；否则原首行首列自带 0 的信息会被后续覆盖弄混。
2. **内层打标仅从 `(1,1)` 开始**，避免一步就把整行首格写 0，误把「行标记」扩散成「首列本来就要清」的歧义（与 flag 分工配合）。
3. **清行 / 清列顺序**：先按首列标记清行、再按首行标记清列（或对称实现），**最后**再处理第 0 行与第 0 列，保证标记区仍可读。
