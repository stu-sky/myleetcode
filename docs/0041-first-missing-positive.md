# 41. 缺失的第一个正数

**代码**：[codes/0041-first-missing-positive.go](../codes/0041-first-missing-positive.go)

## 题目

给你一个未排序的整数数组 `nums`，请你找出其中没有出现的最小的**正整数**。

请你实现时间复杂度为 **O(n)** 并且只使用常数级别的额外空间的解决方案。

**示例**：

- 输入：`nums = [1,2,0]` → 输出：`3`
- 输入：`nums = [3,4,-1,1]` → 输出：`2`
- 输入：`nums = [7,8,9,11,12]` → 输出：`1`

## 思路

### 知识点：鸽笼原理 + 符号打标（原地「哈希」）

长度为 `n` 时，**答案只可能在 `1`～`n+1`**。关心的是：`1`、`2`、…、`n` 里谁没出现——正好可以和下标 **`0..n-1`** 一一对应。  
在不能用额外集合的前提下，用**符号**：若正数 **`k`** 出现过，就把 **`nums[k-1]` 变成负数**，表示「`k` 这个坑有人来过了」；没被标成负的坑，对应缺席的最小正数。打标时需用**绝对值**读原数值，免得把上一轮已经打成负数的格子误判。

### 怎么想到

- **题目在问什么**：最小缺失正整数。
- **朴素做法卡在哪**：排序太慢；哈希对拍最顺但要多 O(n) 空间。
- **换什么技巧**：仍用「答案 ∈ [1,n+1]」把信息压进原数组：**先清掉不配占坑的数**（≤0、>n 改成占位），再「看见 `x` 就给第 `x` 个坑打勾」，最后用**正负号扫一遍**。面试口述可串成三遍循环、每遍一句话，比重置换的 **`i` 走不走**更易记。

### 核心步骤

1. **清洗**：对每个 `nums[i]`，若 `nums[i]≤0` 或 `nums[i]>n`，改为 **`n+1`**（表示「不配参与打标」，且与任意合法 `x∈[1,n]` 区分）。
2. **打标**：遍历每个格子，取 **`x = abs(nums[i])`**。若 **`x≤n`** 且 **`nums[x-1]>0`**，令 **`nums[x-1]=-nums[x-1]`**（只打一次标，重复 `x` 不重复翻转）。
3. **找答案**：从左找第一个 **`nums[i]>0`** 的位置，返回 **`i+1`**；若全为负，返回 **`n+1`**。

### 复杂度

- **时间**：三遍线性扫描 → O(n)。
- **空间**：O(1) 额外。

## 易错点

1. **打标前必须用 `abs(nums[i])` 读出「原本要标记谁」**：否则负数格子的语义会搞混。
2. **仅当 `nums[x-1]>0` 才取负**：避免同一个坑被正反翻两次等于没标（重复元素场景）。
3. **清洗占位用 `n+1`**：`[1,n]` 内才是有效键；`0`、`负数`、**`>n`** 都无法对应「缺失候选」里的坑。
4. 若题述禁止破坏原数组，需事先说明解法假设；**力扣本题允许原位修改数组**。
