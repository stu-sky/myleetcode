# 15. 三数之和

**代码**：[codes/0015-3sum.go](../codes/0015-3sum.go)

## 题目

给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。返回所有和为 0 且**不重复**的三元组。

**示例**：
- 输入：`nums = [-1,0,1,2,-1,-4]`
- 输出：`[[-1,-1,2],[-1,0,1]]`

## 思路

### 问题本质

找三个数和为 0，且结果不能重复。暴力三重循环 O(n³) 太慢，去重也麻烦。

### 核心转化

**3Sum → 固定一个数 + 2Sum**。

排序后，固定 `nums[i]`，问题变成：在 `[i+1, n-1]` 中找两个数使得和为 `-nums[i]`。这就是有序数组上的 2Sum，用**对撞双指针** O(n) 解决。

外层遍历 O(n) × 内层双指针 O(n) = O(n²)。

### 去重策略

排序之后，相同的数会相邻，去重就变得简单：

1. **第一个数去重**：`nums[i] == nums[i-1]` 时跳过（同一个值不重复作为第一个数）
2. **找到答案后去重**：跳过 left 和 right 处相同的值，再各移一步

### 剪枝

- `nums[i] > 0` 时直接 break：排序后最小值已 > 0，三个正数不可能和为 0

### 示例

```
nums = [-1, 0, 1, 2, -1, -4]
排序 → [-4, -1, -1, 0, 1, 2]

i=0, nums[i]=-4, target=4
  L=1(-1), R=5(2): sum=-1+2=1 < 4 → L++
  L=2(-1), R=5(2): sum=-1+2=1 < 4 → L++
  L=3(0),  R=5(2): sum=0+2=2  < 4 → L++
  L=4(1),  R=5(2): sum=1+2=3  < 4 → L++
  L >= R → 结束

i=1, nums[i]=-1, target=1
  L=2(-1), R=5(2): sum=-1+2=1 == 1 ✓ → 记录 [-1,-1,2]
    去重后 L=3, R=4
  L=3(0), R=4(1): sum=0+1=1 == 1 ✓ → 记录 [-1,0,1]
    L=4, R=3 → 结束

i=2, nums[i]=-1, 与 i=1 相同 → 跳过

i=3, nums[i]=0 > ... 继续但找不到和为 0 的组合

结果：[[-1,-1,2], [-1,0,1]]
```

### 复杂度

- 时间 O(n²)：外层 O(n)，内层双指针 O(n)
- 空间 O(log n)：排序栈空间

## 易错点

1. **去重时机**：找到答案后先跳过重复值，再 `left++; right--`。如果顺序反了会漏跳或越界。
2. **第一个数的去重**：是 `nums[i] == nums[i-1]` 而不是 `nums[i] == nums[i+1]`，否则会跳过有效组合（例如 `[-1,-1,2]`）。
3. **剪枝条件**：`nums[i] > 0` 用 `break` 而非 `continue`，因为排序后后续更大，不可能为 0。

## 变种思路

| 变种 | 思路 |
|------|------|
| **两数之和（1）** | 哈希表 O(n)，或排序后对撞双指针 O(n log n) |
| **四数之和（18）** | 再套一层循环：固定两个数 + 对撞双指针，O(n³) |
| **最接近的三数之和（16）** | 同框架，把 `== 0` 改为记录与 target 差值最小的 sum |
