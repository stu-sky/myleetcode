# 76. 最小覆盖子串

**代码**：[codes/0076-minimum-window-substring.go](../codes/0076-minimum-window-substring.go)

## 题目

给定两个字符串 `s` 和 `t`，返回 `s` 中涵盖 `t` 所有字符的**最小子串**（子串中同一种字符个数不能少于 `t` 中该字符个数）。若不存在，返回空串。

**示例**：
- 输入：`s = "ADOBECODEBANC", t = "ABC"` → 输出：`"BANC"`
- 输入：`s = "a", t = "a"` → 输出：`"a"`
- 输入：`s = "a", t = "aa"` → 输出：`""`

## 思路

### 本题会用到的概念（各说一句）

- **滑动窗口**：用左右端点界定 `s` 上的一个连续区间，右端右移「扩大」窗口，左端右移「缩小」，在「可行」与「尽量短」之间调整。
- **哈希计数**：`need[c]` 表示字符 `c` 在 `t` 里要出现几次；`window[c]` 表示当前窗口里 `c` 出现了几次。不从头扫整个 `t` 对比，而是用计数在 O(1) 内判断「是否覆盖」。

---

### 怎么想到

1. **题目在问什么**：在 `s` 里找一个**最短**连续区间，使得 `t` 里每种字符的频次都被满足（可多于 `t`，不能少于）。
2. **暴力**：枚举所有子串 O(n²)，每个检查是否覆盖 `t` 再 O(|t|)，总体太慢。
3. **卡在哪**：检查「是否覆盖」可以改成**与频次比对**，不必每次重扫 `t`；短板是枚举所有区间太多。
4. **突破口**：若固定左端点，右端点向右扩展时，窗口只会**多包含**字符；一旦已经覆盖 `t`，再右扩只会让窗口变长，对「求最短」无益——此时应尝试**左端右移缩小窗口**，直到不再覆盖，再让右端继续扩。这是经典的**可变长滑动窗口**：**右扩求可行，左缩求最优（最短）**。

---

### 怎么做

维护 `need`（`t` 的需求）、`window`（当前窗口内计数）。

再维护 **`valid`**：有多少种字符 `c`（在 `need` 里出现的）已经满足 `window[c] == need[c]`。当 `valid == len(need)` 的 key 种类数（即 `t` 中不同字符种类数，记为 `needTypes`）时，窗口**合法**。

每步：

1. **`right` 右移**，把 `s[right]` 纳入 `window`；若该字符在 `need` 里且 `window` 从 `need[c]-1` 增到 `need[c]`，`valid++`。
2. **`while valid == needTypes`**：当前窗口已覆盖 `t`，更新最短答案；然后 **`left` 右移**尝试变短，移出 `s[left]` 时若在 `need` 里且移出前 `window[d]==need[d]`，则 `valid--`，再 `window[d]--`。

结束后若无合法窗口，返回 `""`。

---

### 复杂度

- 时间 O(|s| + |t|)：`left`、`right` 各最多走 |s| 步
- 空间 O(|字符集|)：本题为字母，可视为常数级

## 易错点

1. **`valid` 的含义**：只对满足 `need[c]>0` 的字符，在 `window[c]==need[c]` 时贡献 1；不是窗口里字符种类数。
2. **左缩时先判再减**：若 `window[d]==need[d]`，移出前要先 `valid--`，再 `window[d]--`。
3. **`needTypes`**：是 `need` 的键个数（有几种字符要凑齐），不要用 `len(t)`。
