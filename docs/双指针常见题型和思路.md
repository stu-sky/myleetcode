# 双指针常见题型和思路

## 什么时候该想到双指针？

遇到以下关键词或特征时，应优先考虑双指针：

| 信号 | 举例 |
|------|------|
| **原地操作**，不能用额外数组 | 移动零、移除元素 |
| **有序数组**上查找一对/多个元素 | 两数之和、三数之和 |
| **连续子串/子数组**满足某条件 | 最长无重复子串、最小覆盖子串 |
| **链表**找中点、找环、找倒数第 k 个 | 环形链表、链表中间结点 |
| **回文**判断 | 验证回文串 |
| **容器/面积**类问题 | 盛最多水的容器、接雨水 |

---

## 五种常见模式

### 模式一：发现 + 写入（同向，不同步）

**指针含义**：
- `right`：遍历指针，负责扫描每个元素
- `left`：写入指针，标记下一个要写入的位置

**移动规则**：`right` 每轮都动，`left` 只在满足条件时前进。

**适用场景**：原地删除、整理、去重。

**模板**：

```go
left := 0
for right := 0; right < len(nums); right++ {
    if 满足保留条件(nums[right]) {
        nums[left] = nums[right] // 或交换
        left++
    }
}
// [0, left) 是整理后的结果
```

**典型题目**：

| 题号 | 题目 | 保留条件 |
|------|------|----------|
| 283 | 移动零 | `nums[right] != 0` |
| 27 | 移除元素 | `nums[right] != val` |
| 26 | 删除有序数组中的重复项 | `nums[right] != nums[left-1]` |

---

### 模式二：两端向中间（对撞指针）

**指针含义**：
- `left`：从最左端开始
- `right`：从最右端开始

**移动规则**：根据当前结果与目标的比较，决定移动哪一侧。`left < right` 时继续。

**适用场景**：有序数组上找一对元素、对称性判断、面积类问题。

**模板**：

```go
left, right := 0, len(nums)-1
for left < right {
    if 条件偏小 {
        left++
    } else if 条件偏大 {
        right--
    } else {
        // 找到目标
    }
}
```

**典型题目**：

| 题号 | 题目 | 移动依据 |
|------|------|----------|
| 167 | 两数之和 II | 和 < target → left++，和 > target → right-- |
| 11 | 盛最多水的容器 | 移动较矮的一端 |
| 125 | 验证回文串 | 两端字符不等 → 不是回文 |
| 15 | 三数之和 | 固定一个数，对剩余部分用对撞指针 |

---

### 模式三：快慢指针（同向，不同速度）

**指针含义**：
- `slow`：慢指针，每次走 1 步
- `fast`：快指针，每次走 2 步（或 k 步）

**移动规则**：两个都向前，但步长不同。

**适用场景**：链表找环、找中点、找倒数第 k 个。

**模板（找环）**：

```go
slow, fast := head, head
for fast != nil && fast.Next != nil {
    slow = slow.Next
    fast = fast.Next.Next
    if slow == fast {
        // 有环
    }
}
```

**典型题目**：

| 题号 | 题目 | 用法 |
|------|------|------|
| 141 | 环形链表 | slow 和 fast 相遇 → 有环 |
| 142 | 环形链表 II | 相遇后，一个回到 head，同步走 → 找入口 |
| 876 | 链表中间结点 | fast 到底时，slow 在中点 |
| 19 | 删除倒数第 N 个结点 | fast 先走 N 步，再同步走 |

---

### 模式四：滑动窗口（同向，维护区间）

**指针含义**：
- `left`：窗口左边界
- `right`：窗口右边界

**移动规则**：`right` 向右扩展窗口，当窗口不满足条件时，`left` 向右收缩。

**适用场景**：连续子串/子数组满足某种约束（最长/最短/恰好）。

**模板**：

```go
left := 0
for right := 0; right < len(s); right++ {
    // 1. 将 s[right] 加入窗口

    for 窗口不满足条件 {
        // 2. 将 s[left] 移出窗口
        left++
    }

    // 3. 更新答案（最长/最短取决于题意）
}
```

**典型题目**：

| 题号 | 题目 | 窗口条件 |
|------|------|----------|
| 3 | 无重复字符最长子串 | 窗口内无重复字符，取最长 |
| 76 | 最小覆盖子串 | 窗口包含目标所有字符，取最短 |
| 438 | 找所有字母异位词 | 窗口内字符频次 = 目标频次 |
| 209 | 长度最小的子数组 | 窗口内和 ≥ target，取最短 |

---

### 模式五：区间划分（多指针分区）

**指针含义**：
- 多个指针把数组分成若干段，每段有不同含义

**适用场景**：将数组按条件分成 2~3 个区域。

**模板（以三分区为例）**：

```go
lo, mid, hi := 0, 0, len(nums)-1
for mid <= hi {
    switch {
    case nums[mid] < pivot:
        nums[lo], nums[mid] = nums[mid], nums[lo]
        lo++
        mid++
    case nums[mid] > pivot:
        nums[mid], nums[hi] = nums[hi], nums[mid]
        hi--
    default:
        mid++
    }
}
```

**典型题目**：

| 题号 | 题目 | 分区方式 |
|------|------|----------|
| 75 | 颜色分类 | 三指针：0 区、1 区、2 区 |

---

## 总结：遇到双指针题怎么想？

**三步法**：

**第一步：识别模式**

| 题目特征 | 对应模式 |
|----------|----------|
| 原地整理/去重/删除 | 发现 + 写入 |
| 有序 + 找一对/配对 | 对撞指针 |
| 链表 + 环/中点/倒数 | 快慢指针 |
| 连续子串/子数组 + 最长最短 | 滑动窗口 |
| 分区/分类 | 多指针划分 |

**第二步：定义指针**

明确每个指针**代表什么**。不是随便两个下标，而是有明确职责的变量。

**第三步：定移动规则**

什么时候动哪个指针、动多少、什么条件下停。

> **核心原则**：先定义清楚两个指针各自的含义，代码自然就出来了。定义不清，怎么写都会乱。
