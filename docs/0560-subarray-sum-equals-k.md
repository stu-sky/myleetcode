# 560. 和为 K 的子数组

**代码**：[codes/0560-subarray-sum-equals-k.go](../codes/0560-subarray-sum-equals-k.go)

## 题目

给你一个整数数组 `nums` 和一个整数 `k`，请你统计并返回该数组中和为 `k` 的**连续子数组**的个数。

**示例**：
- 输入：`nums = [1,1,1], k = 2` → 输出：`2`（子数组 `[1,1]` 与 `[1,1]`，注意重叠也算）
- 输入：`nums = [1,2,3], k = 3` → 输出：`2`（`[1,2]` 与 `[3]`）

## 思路（用人话讲）

### 先想暴力

枚举每个起点、每个终点，算区间和是不是 `k`。太慢，O(n²)。

---

### 约定几个量（后面都用同一套说法）

- **`sum`**：从数组**下标 0** 一直加到**当前下标**（含）的累加和，也就是「前缀和」。
- **`cnt`**：**哈希表**，用来回答这个问题——  
  **「在遍历到当前元素之前，前缀和等于某个值 `t` 的情况，一共出现过几次？」**  
  - **键**：前缀和的数值 `t`  
  - **值**：这个 `t` 作为「某一段前缀的结尾和」已经出现过几次  

  换句话说：`cnt[t]` = 之前有多少个位置，使得「从开头加到那个位置」的和恰好是 `t`。  
  代码里每处理完一个位置，就把当前的 `sum` 记进表：`cnt[sum]++`。

---

### 子数组和与前缀和的关系（推一遍等式）

设子数组是 **`nums[i]` 到 `nums[j]`**（都包含），和为 `k`。

- 从开头加到 `j` 的前缀和：`sum` = `nums[0] + … + nums[j]`
- 从开头加到 `i-1` 的前缀和：记为 `prev` = `nums[0] + … + nums[i-1]`

子数组的和 = 大前缀 − 小前缀：

> `sum − prev = k`

移项得到：

> **`prev = sum − k`**

所以要找的**不是**「`k` 减 `sum`」，而是「**当前前缀和 `sum` 减去目标和 `k`**」——也就是**子数组左边界左边**那一小段前缀和必须是 **`sum - k`**。

若用 `k - sum` 去查表，含义变成「之前某处前缀和是 `k - sum`」，代回式子会得到错误的区间关系（不是「中间一段和为 `k`」）。

---

### 算法在做什么（接上 `cnt`）

每走到一个新下标：

1. 把当前数加进 `sum`（更新「到当前为止」的前缀和）。
2. **`ans += cnt[sum - k]`**  
   含义：之前有多少次，前缀和刚好是 `sum - k`？  
   每出现一次，就对应**一种**左边界选择，使得「从左边界到当前」这一段的和是 `k`。
3. **`cnt[sum]++`**  
   把「当前这个前缀和」记入表，供后面的位置使用。

注意顺序：先查 `cnt[sum-k]`，再 `cnt[sum]++`，避免把**当前位置**当成子数组的左边界前面的前缀（子数组必须从当前或更早结束，左边界必须在当前之前）。

---

### 为什么要先 `cnt[0] = 1`？

子数组可能**从下标 0 开始**。此时「左边界左边」没有元素，相当于那一段前缀和是 **0**，且应算作 **出现过 1 次**。  

初始化 `cnt[0] = 1` 后，当第一次 `sum == k` 时，查的是 `cnt[sum - k] = cnt[0] = 1`，能正确数到「从开头到当前」这一整段。

**若误用 `k - sum` 举例**：设 `sum = 10, k = 3`，正确要找的是「前面前缀和 = 7」，因为 `10 - 7 = 3`。若去查 `k - sum = -7`，完全不是「左边界前的和」，和「中间一段和为 k」对不上。

### 小例子

`nums = [1,2,3], k = 3`（`cnt` 记录「历史前缀和」出现次数）

| 步骤 | 当前数 | sum | 查 cnt[sum-k] | ans | 含义 |
|------|--------|-----|---------------|-----|------|
| 初值 | - | 0 | - | 0 | 只初始化了 `cnt[0]=1`，表示空前缀和为 0 算 1 次 |
| +1 | 1 | 1 | cnt[-2]=0 | 0 | 再 `cnt[1]=1` |
| +2 | 2 | 3 | cnt[0]=1 | 1 | 说明之前有 1 次前缀和为 0，对应子数组 [1,2] |
| +3 | 3 | 6 | cnt[3]=1 | 2 | 之前有 1 次前缀和为 3，对应子数组 [3] |

### 复杂度

- 时间 O(n)：每个元素处理一次
- 空间 O(n)：哈希表里不同前缀和的个数

## 易错点

1. **`cnt[0] = 1` 不能忘**，否则从开头开始的合法子数组会漏算。
2. **顺序**：先 `ans += cnt[sum-k]`，再 `cnt[sum]++`。若先加当前 `sum`，会把「当前位置当左端点」的错误情况算进去。
3. **数组里有负数**：仍然成立，前缀和可能上下波动，哈希表照样统计即可。

## 变种思路

| 变种 | 思路 |
|------|------|
| **最长/最短和为 k 的子数组** | 仍用前缀和，配合记录每个前缀和**第一次/最后一次**出现的下标 |
| **二维矩阵里和为 k 的子矩阵** | 枚举上下边界，压成一维，再套本题的一维做法 |
| **乘积小于 k 的子数组** | 不是加法前缀和，通常用双指针（正数数组） |
