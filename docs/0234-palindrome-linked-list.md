# 234. 回文链表

**代码**：[codes/0234-palindrome-linked-list.go](../codes/0234-palindrome-linked-list.go)

题库入口：[234. 回文链表](https://leetcode.cn/problems/palindrome-linked-list/description/?envType=study-plan-v2&envId=top-100-liked)

## 题目

给你一个单链表的头节点 `head`，请判断该链表是否为回文链表。若是返回 `true`；否则返回 `false`。

进阶：能否用 **\(O(n)\)** 时间与 **\(O(1)\)** 额外空间解决？

链表节点：`type ListNode struct { Val int; Next *ListNode }`。函数：`func isPalindrome(head *ListNode) bool`。空链可视为回文。

**示例**：

- `[1,2,2,1]` → `true`  
- `[1,2]` → `false`

## 思路

### 知识点：快慢指针定中点；反转后半段链表

**快慢指针**：两块指针一起走，快指针每步跨过两个结点、慢指针每步跨一个结点；当快到尾时慢大约落在链表**中点附近**——用来把一个「对折」对称问题拆开成两半，而不必新开 \(O(n)\) 切片存整链。**链表反转**：把某一段 `next` 全部逆转方向，本题用来把后半段「掉个头」，这样就能与从头开始的前半段**同向逐项比较**。反转用三指针迭代即可。

### 怎么想到

- **题目在问什么**：从左读和从右读的结点值序列对称。链表不能随机访问右端「对称位」。  
- **朴素卡在哪**：复制到切片再首尾双指针可行，进阶要求 **不写额外线性结构**。  
- **换什么技巧**：链表回文本质是 **前半 ≡ 翻转后的后半**。先在不走 \(O(n)\) 数组的前提下锁定**前半尾的下一个就是后半的起点**，翻转后半再走一遍逐项比；比完最好能**恢复原链**，面试里显得更干净。

### 核心步骤

1. **`endOfFirstHalf`**：`fast.Next` 连续两次还能走时循环；结束后 **`slow` 为「左半只 + 中点」路径上的最后一格**——**偶数**时即左半截的**尾结点**；**奇数**时 `slow` 落在**正中间那个结点**（对称值只比两侧，不比中间本身，故反转从 `slow.Next` 开始即可）。  
2. **`secondHalfHead := reverse(slow.Next)`**：只反转 **后半**。  
3. **`p1` 从 head、`p2` 从 `secondHalfHead` 同步走**，对每个 `p2 != nil` 对齐比较 `Val`（奇数个结点时中段只会出现在单侧，比较在 `p2` 用尽时停止，自然不会误比）。  
4. **再反转一次后半**接回原位，恢复原结构。  

### 复杂度

- **时间**：找中点、反转、比较皆线性扫链 → **\(O(n)\)**。  
- **空间**：仅若干指针 → **\(O(1)\)**。

### 易错点

1. **`while fast != nil && fast.Next != nil` 与本题写法不等价**：另一套常使 `slow` 落在「第二个中点」；本题用 **`fast.Next != nil && fast.Next.Next != nil`** 是为了让 **`slow.Next`** 稳定地作为**待反转的后半头**，并与「奇数时 `slow` 在中点、反转不含中点」这一套配合，不要两套混用却忘了改反转移位。  
2. **比较条件是 `for p2 != nil`**（不是硬数一半个数），可同时照顾奇偶长度与中点不参与第二次比较。  
3. **恢复原链**：若评测或后续还用原链表，`secondHalfHead` 要保留，最后用同一段反转移回去。单机提交力扣不写恢复有时也能过系统用例，但面试建议说明并写回。
