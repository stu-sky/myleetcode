# 128. 最长连续序列

**代码**：[codes/0128-longest-consecutive-sequence.go](../codes/0128-longest-consecutive-sequence.go)

## 题目

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

**示例**：
- 输入：`nums = [100,4,200,1,3,2]`
- 输出：`4`（最长连续序列是 [1,2,3,4]）

## 思路

### 问题本质

在无序数组中找**最长连续整数序列**（如 1,2,3,4），要求 O(n) 时间。  
排序会变成 O(n log n)，不符合要求。因此需要**用哈希在 O(1) 内判断某数是否存在**。

### 解法：只从序列起点扩展

若从每个数都向上扩展（x, x+1, x+2...），同一段会被重复遍历。  
**优化**：只有当 `x-1` 不在集合中时，才认为 `x` 是序列起点，从 `x` 开始向上扩展。这样每个连续段只会被遍历一次。

### 示例

```
nums = [100, 4, 200, 1, 3, 2]
set = {1, 2, 3, 4, 100, 200}

100: 99 不在 set → 起点，扩展 100 → 长度 1
4:   3 在 set   → 非起点，跳过
200: 199 不在   → 起点，扩展 200 → 长度 1
1:   0 不在     → 起点，扩展 1,2,3,4 → 长度 4 ✓
3:   2 在       → 跳过
2:   1 在       → 跳过

答案：4
```

### 复杂度

- 时间 O(n)：遍历 set（去重后），每个数最多参与两次（一次判断是否起点，一次在扩展时被访问）
- 空间 O(n)：哈希集合

## 易错点

1. **遍历 set 而非 nums**：若有重复元素，遍历 nums 会从同一起点多次扩展，退化为 O(n²)；应遍历 set 保证 O(n)。
2. **空数组**：返回 0，循环不执行时 maxLen 保持 0 即可。
3. **为何不删 set 中的元素**：删不删不影响正确性；不删更简单，且每个数最多被扩展时访问一次，仍为 O(n)。

## 变种思路

| 变种 | 思路 |
|------|------|
| **返回序列本身** | 扩展时记录起点和终点，最后返回 `[start, end]` 或 `[start, start+1, ..., end]` |
| **允许重复** | 若重复算多次，需用 `map[int]int` 记频次，扩展时累加 |
| **二维连续** | 如矩阵中相邻格，可用并查集或 BFS 找连通块 |
