# 48. 旋转图像

**代码**：[codes/0048-rotate-image.go](../codes/0048-rotate-image.go)

说明：题库中「`/description/`」与根路径两道入口指向**同一道题**：[48. 旋转图像](https://leetcode.cn/problems/rotate-image/)。

## 题目

给定一个 **n × n** 的二维矩阵 `matrix`（表示一幅图像），将图像 **顺时针旋转 90°**。要求 **`原地`** 修改，即不要使用另一块 `n × n` 缓冲矩阵作答（仅允许常量级额外变量）。

**示例**：

- 输入：`matrix = [[1,2,3],[4,5,6],[7,8,9]]`  
  输出：`[[7,4,1],[8,5,2],[9,6,3]]`（对 `matrix` 原地改写）
- 输入：`matrix = [[5,1,9,11],[2,4,8,10],[13,3,6,7],[15,14,12,16]]`  
  输出：`[[15,13,2,5],[14,3,4,1],[12,6,8,9],[16,7,10,11]]`

## 思路

### 知识点：矩阵的「线性变换」分拆成两次简单改写

顺时针 90° 可以看成对下标做一次固定规则的重排。**转置**：把矩阵沿主对角线翻过去，等价于 **`matrix[i][j]` 与 `matrix[j][i]`** 互换（只当上三角 **`j > i`** 时做一次，避免交换两次抵消）。再对**每一行做左右镜像**（首尾两两互换），合在一起就是整张图顺时针 90°；这两步都只交换元素位置、不复制整块矩阵，故满足原地要求，且口述时容易拆成「先对角、再横行翻转」两个阶段。

### 怎么想到

- **题目在问什么**：在 **O(1)** 额外空间的约束下完成一次全局重排。
- **朴素卡在哪**：若按「算出目标格 `(i,j)` 该放哪个源值」逐项抄到新区间，往往需要辅助数组；单次四元轮换的置换环写法也能做，但对白时步骤不如「两遍简单变换」规整。
- **为何用转置 + 行翻转**：这是固定 **n × n** 下与顺时针 90° 等价的经典分解，每层循环语义单一，边界只有「转置只用上三角」「每行翻转只到 **`n/2`**」两处。

### 核心步骤

1. **转置**：`i` 从 `0` 到 `n-1`，内层 **`j := i + 1` 到 `n-1`**，交换 `matrix[i][j]` 与 `matrix[j][i]`。  
2. **每行镜像**：对每个 `i`，`j` 从 **`0` 到 `n/2 - 1`**，交换 `matrix[i][j]` 与 `matrix[i][n-1-j]`。

### 复杂度

- **时间**：每个元素参与常数次交换 → **O(n²)**。
- **空间**：仅若干下标变量 → **O(1)** 额外（输出仍写在原 `matrix` 上）。

### 易错点

1. **转置**时对 **`j <= i`** 也对调会抵消，必须限制为 **`j > i`**（或等价写法）。  
2. **行翻转**内层 **`j < n/2`**，不要用错成 `n-1`。  
3. **`n == 1`**：合法输入，可无操作返回。

另有主流写法：**按层**从外到内，对每条边上若干位置做 **四角元素环状轮换**（注意内层 **`j`** 上界常为 **`n-2-i`**)，与本题结果等价；面试若更熟悉置换式索引再选用即可。
