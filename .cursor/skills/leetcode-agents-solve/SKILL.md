# LeetCode 本题仓库解题（AGENTS）

## 首要依据

执行前先阅读仓库根目录 [AGENTS.md](../../../AGENTS.md)。本 skill 是摘要；与 AGENTS 冲突时 **以 AGENTS.md 为准**。

## 何时写文件

遵循 AGENTS「行为边界」：

- 用户**仅要思路/讲解**：优先**对话中**文字（可含代码块），**不要**主动新建或修改 `docs/`、`codes/`。
- 用户**明确要求**写入仓库（如「输出文档」「写到 docs」「按 AGENTS 生成」或附带题号要写进项目）：再创建或更新 `docs/` 与 `codes/` 下对应文件。
- **与上条配合**：用户给出 **力扣题目链接或题号** 并索要「解法、怎么做、优雅写法」等，且**未**写明「只要口头 / 不写文件」时，**默认**按本 skill **生成或更新成对的 `docs/` + `codes/`**，与「泛泛问某一类题型」区分开。

## 命名（强制）

统一 `{LeetCode编号}-{英文题名-kebab-case}`：

| 类型 | 路径示例 |
|------|-----------|
| Go | `codes/0238-product-of-array-except-self.go` |
| 文档 | `docs/0238-product-of-array-except-self.md` |

编号 **四位前补零**（`1` → `0001`）。

## Go 代码约定

- **保持力扣原函数签名**，便于粘贴提交。
- **只保留一个解法**（不并列多种）。**解法选取见下「面试取向」**；仍须满足题的时空要求。

## 面试取向：优先「最好讲、最好记」

本仓库题目面向面试口述与复习，**在时间空间同阶、均满足题设的多种写法里**，默认选：

1. **更好分成「三遍在干什么」的步骤**（如先清洗再打标再找答案），少用「外层 `for` + 内层 `while`/交换且不移动指针」的技巧型写法——除非题目明显卡常或声明要极简。
2. **口头模板清晰**优先于代码行数；**「看见 k 就给第 k 个坑记一笔」类标记法**，常优于等价但易在边界/死循环上卡壳的置换法（典型题：`firstMissingPositive`、`findDisappearedNumbers`、442 类等）。
3. **若只有一种达标写法**，照写；确有折中再在文档「易错点 / 尾部备注」里**一句**带过其它主流写法即可，不写第二份完整代码。
4. 若用户写明「要最短代码」「要某种模板」，以用户为准。


## 文档 `docs/*.md` 结构（强制块）

1. 标题：`# {编号}. {中文或英文题名}`。
2. **代码链接**（置顶，链接格式照抄）：
   ```markdown
   **代码**：[codes/XXXX-slug.go](../codes/XXXX-slug.go)
   ```
3. **题目**：题干、输入输出、**示例**（复习可不打开网页）。
4. **思路**：含下面「叙述风格」三条。
5. **复杂度**：时间与空间（注明输出数组是否计入额外空间等题面说法）。
6. **易错点**：列表。
7. **变种思路**：见下节。

## 题解叙述风格（与 AGENTS 一致）

1. **知识点首次「点一句」**：第一次出现双端队列、前缀和、滑动窗口、单调栈/单调队列、哈希计数等，用 **2～4 句话** 说清是什么、在本题里干什么；不讲成大部头教程。
2. **先「怎么想到」再「怎么做」**：一小节（如「怎么想到」「思考路径」）：**题目在问什么 → 朴素哪里卡（时间/空间/信息）→ 为何换数据结构/技巧**。忌一上来只列步骤。
3. **控制总长**：思路主体 **一到两屏**；短段落 + 列表；与 1、2 冲突时 **删重复与次要细节**，不牺牲 1、2。

## 变种思路一节

- **必须**列 **同类力扣题**（题号 + 短题名 + **一句话**与本题关系，如「同左右缀拼答案」「同滑动窗口」）。
- 可补充可对标的抽象变种；**表格或短列表**，不写成第二篇题解；**无合适题则不凑数**。

## 执行清单（写文件时）

- [ ] 文件名与编号、题名 slug 一致（Go 与 md 成对）。
- [ ] 文档含题目全文要点 + 示例 + 代码相对链接。
- [ ] 思路含：知识点一句、思考路径、核心步骤、复杂度、易错点、变种表。
- [ ] Go 仅一解、签名与力扣一致。
- [ ] **面试取向**：多解同阶时选对白最顺的版本；文末可一句对比其它主流写法。

## 技术栈提示

- Go 1.20+；**UTF-8**。
